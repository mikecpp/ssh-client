# SSH Web Client

A web-based SSH client built with Golang backend and React frontend.

## Features

- Web-based terminal interface using xterm.js
- Password authentication for SSH servers
- Real-time terminal I/O via WebSocket
- Responsive terminal UI with automatic resizing

## Architecture

- **Backend**: Golang with WebSocket server for SSH connections
- **Frontend**: React with Vite and xterm.js terminal emulator
- **Communication**: WebSocket protocol for real-time data transfer

## Prerequisites

- Go 1.16 or higher
- Node.js 14 or higher
- npm or yarn

## Installation

1. Clone the repository:
```bash
git clone <repository-url>
cd ssh-client
```

2. Install backend dependencies:
```bash
cd backend
go mod download
```

3. Install frontend dependencies:
```bash
cd ../frontend
npm install
```

## Running the Application

### Development Mode

1. Start the backend server:
```bash
cd backend
go run main.go
```

The server will start on port 5000.

2. In a new terminal, start the Vite development server:
```bash
cd frontend
npm run dev
```

The frontend will open in your browser at http://localhost:5173

### Production Mode

1. Build the frontend:
```bash
cd frontend
npm run build
```

2. Start the backend server (it will serve the built frontend):
```bash
cd backend
go run main.go
```

3. Open your browser and navigate to:
```
http://localhost:5000
```

## Usage

1. Enter your SSH server details:
   - Host: SSH server hostname or IP address
   - Port: SSH port (default: 22)
   - Username: SSH username
   - Password: SSH password

2. Click "Connect" to establish an SSH session

3. Use the terminal to interact with your remote server

## Security Notes

**WARNING**: This application is intended for development and testing purposes only.

- The current implementation uses `ssh.InsecureIgnoreHostKey()` which skips host key verification
- Passwords are transmitted over WebSocket (use HTTPS/WSS in production)
- For production use, you should:
  - Implement proper host key verification
  - Use WSS (WebSocket Secure) with TLS
  - Add authentication/authorization for the web interface
  - Consider using SSH key-based authentication instead of passwords
